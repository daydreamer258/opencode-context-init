---

# /forget — 删除记忆条目

支持原则 9（用户驱动写入/删除）和原则 11（陈旧条目应当被删，否则会误导 Agent）。

---

## Phase 0：前置检查

1. 检查 `AGENTS.md` 是否存在。**不存在**：告知"还没有记忆库"，退出。
2. 获取删除目标：
   - 用户给了 ID（`G-003` / `M-007` / `<file>:M-007`）：精确定位
   - 用户给了关键词：进入 Phase 1 搜索

---

## Phase 1：定位条目

### 1.1 按 ID 定位

- 跨 `docs/agent/*.md` 和 `AGENTS.md` 搜索条目 ID
- 找到：跳到 Phase 2
- 找不到：告知"未找到 ID `<X>`，是否换关键词搜索？"

### 1.2 按关键词搜索

扫描所有条目的标题、事实、标签，列出匹配项：

```
按关键词 "<keyword>" 找到 N 条：

  1. docs/agent/auth.md / M-003: OAuth refresh 并发死锁
     标签: oauth, race-condition
  2. docs/agent/_global.md / G-007: 错误码必须用 ErrorCode 枚举
     标签: error-handling, conventions
  ...

要删哪一条？（编号 / 取消）
```

---

## Phase 2：展示要删除的内容并二次确认

```
准备删除：

  文件：docs/agent/<文件>
  条目：<ID>: <标题>  [<类型>]
  - 关联代码: <路径>
  - 新增: <日期>
  - 最后验证: <日期>
  - 标签: <标签>

  事实: ...
  为什么: ...
  反例: ...（如有）

⚠️ 这条记忆将被删除（git 仍保留历史，可回滚）。

确认（y / 取消）？
```

---

## Phase 3：删除

1. 从目标文件中**整段移除**该条目（包括 `### <ID>: 标题` 标题行到下一个 `---` 分隔符之间的所有内容）
2. **不要重排其他条目的 ID**——保持稳定，否则其他地方引用会失效
3. 检查并更新目标文件 frontmatter 的"最后更新"为今日日期
4. 如果删完后某章节为空（如"陷阱与历史坑"下所有条目都没了），**保留章节标题**（不要删空标题）
5. 输出确认：

```
✓ 已从 docs/agent/<文件> 删除条目 <ID>

提醒：
- git diff 可以看具体删了什么
- 误删可用 git checkout <文件> 恢复
```

6. **不要**自动 git add / commit。

---

## 边界与禁忌

- ❌ 不要在没二次确认时删除
- ❌ 不要重排其他条目 ID
- ❌ 不要删 AGENTS.md 的固定 4 段（元信息 / 能力 / 钩子 / 索引）——这些是结构不是条目
- ❌ 不要一次删多条——用户每次只删一条以避免误伤
- ❌ 不要自动 git 操作
