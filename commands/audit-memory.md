# /audit-memory — 审计陈旧记忆

支持原则 11（衰减与审计）。**陈旧记忆比没有记忆更糟，因为它会误导 Agent。**

本命令是**只读**的——只报告问题，由用户决定用 `/forget` 或 `/update-memory` 处置。

---

## Phase 0：前置检查

1. 检查 `AGENTS.md` 是否存在。**不存在**：告知"还没有记忆库"，退出。
2. 范围：
   - 无参数：全量审计 `AGENTS.md` + `docs/agent/*.md`
   - 给了文件名：只审计该文件

---

## Phase 1：分类扫描

按以下 4 个维度逐条评估每个记忆条目。委托 `memory-auditor` Skill 执行具体扫描（共享逻辑）。

### 1.1 关联代码已消失（高优先级）

如果条目 `关联代码` 字段指向的文件/路径**已不存在**：

→ 标记为 **STALE_CODE**

理由：代码都没了，记忆大概率失效。

### 1.2 时间戳陈旧

如果"最后验证"距今 **超过 6 个月**：

→ 标记为 **STALE_TIME**

理由：长期没被复核，可能已经过时。

### 1.3 内容可推导（违反原则 1）

条目内容看起来能从代码直接读出（如纯描述当前结构、纯列举现有命令）：

→ 标记为 **DERIVABLE**

理由：违反原则 1，占据记忆库价值。

### 1.4 字段不完整

缺少"为什么"且类型是 `decision` 或 `gotcha`：

→ 标记为 **INCOMPLETE_WHY**

理由：未来 Agent 不知道何时该破例。

### 1.5 重复或近重复

跨条目对比，存在事实文本高度重叠的条目：

→ 标记为 **DUPLICATE**

---

## Phase 2：输出报告

按文件 + 严重度组织。**不要**逐条列出全部条目——只列**有问题**的。

```
📋 记忆审计报告

扫描范围：AGENTS.md + docs/agent/*.md
扫描时间：<今日>

────────────────────────────────────────
🔴 高优先级（建议尽快处理）

docs/agent/auth.md:
  - M-003 [STALE_CODE] 关联代码 src/auth/legacy.ts 已不存在
    建议：/forget M-003，或更新代码路径
  - M-007 [DUPLICATE] 与 _global.md G-012 内容重叠
    建议：保留更具体的一条，/forget 另一条

────────────────────────────────────────
🟡 中等（建议复核）

docs/agent/_global.md:
  - G-002 [STALE_TIME] 最后验证: 2025-09-12（距今 8 个月）
    建议：/update-memory G-002 复核并刷新"最后验证"
  - G-005 [DERIVABLE] 内容"项目用 PostgreSQL"可从 docker-compose.yml 推断
    建议：/forget G-005

────────────────────────────────────────
🟢 低（信息缺失，不影响使用）

docs/agent/billing.md:
  - M-002 [INCOMPLETE_WHY] decision 类型但无"为什么"
    建议：/update-memory M-002 补充原因

────────────────────────────────────────

汇总：12 条条目，4 项高优先级、3 项中、1 项低。
```

如果**无问题**：

```
✓ 记忆库 12 条全部健康，无陈旧/重复/缺失字段。
```

---

## Phase 3：用户决策

只输出报告，**不自动处理**。用户看完报告后自己跑：

- `/forget <ID>` 删除
- `/update-memory <ID>` 更新

报告末尾提示：

```
处置建议是供参考——是否执行由你决定。
对单条不确定时可以 /update-memory <ID> 走预览流程，不会立刻丢数据。
```

---

## 边界与禁忌

- ❌ 不要在审计中**自动删除/修改**任何条目（原则 9）
- ❌ 不要漏报——所有命中的问题都要列出
- ❌ 不要谎报"完全健康"如果有问题
- ❌ 不要把 STALE_TIME 阈值（6 个月）当严格规则——只是经验值，提示"建议复核"，不是"必须删"
- ❌ 不要审计 AGENTS.md 的 4 段结构（§ 1–4）——那些不是条目
