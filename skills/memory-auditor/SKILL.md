# Memory Auditor（审计共享逻辑）

被 `/audit-memory` 命令调用的扫描和分类逻辑。

**对应原则：11（衰减与审计）。陈旧记忆比没有记忆更糟。**

---

## 扫描范围

- `AGENTS.md` 的 `§ 5 给 Agent 的元规则` 之外的**条目**（如有内嵌的）
- `docs/agent/*.md` 所有条目（含 `_global.md` 和各模块文档）

不审计：`AGENTS.md` 的 4 段固定结构（§ 1 元信息 / § 2 能力 / § 3 钩子 / § 4 索引），这些是结构不是条目。

---

## 5 类问题与判定规则

### 1. STALE_CODE（高优先级）

**判定**：条目的 `关联代码` 字段指向的文件/路径不存在。

**具体操作**：
- 解析关联代码为路径列表（可能多个，逗号分隔）
- 对每个路径用 Glob 检查存在性
- 若**全部不存在**：标记 STALE_CODE
- 若部分不存在：标记为 STALE_CODE_PARTIAL（中等优先级）

**例外**：路径含 "全局"、"待补充"、"整个仓库" 等占位词时跳过检查。

### 2. STALE_TIME（中等优先级）

**判定**：`最后验证` 距今超过 6 个月。

**具体操作**：
- 解析 `最后验证: YYYY-MM-DD`
- 与今日日期对比
- 超过 180 天：标记 STALE_TIME
- 超过 365 天：标记 STALE_TIME_SEVERE（高优先级）

**说明**：6 个月是经验值，不是硬阈值——只提示"建议复核"。

### 3. DERIVABLE（中等优先级）

**判定**：条目内容可以从代码或常见配置文件直接推导。

**具体操作**：
- 标题或事实文本是否包含以下模式：
  - "项目用 <语言>" / "项目用 <框架>" → 可从 `pom.xml`/`package.json` 推
  - "X 类放在 Y 目录下" → 可从目录结构推
  - "用 X 注入" → 可从 imports 推
  - 纯命令列表（如"构建命令是 mvn package"） → 可从 `Makefile`/`scripts` 推
- 没有"为什么"字段（即只描述现状，不解释原因）
- 类型是 `constraint` 但内容只是描述"代码现在是这样写的"

**注意**：判断 DERIVABLE 要**保守**——拿不准时不标记。

### 4. INCOMPLETE_WHY（低优先级）

**判定**：
- 类型是 `decision` 或 `gotcha`
- 但 `为什么` 字段为空、缺失、或只有"（待补充）"

**说明**：未来 Agent 不知道这个决策的边界，可能误用或破例。

### 5. DUPLICATE（中等优先级）

**判定**：跨条目对比，存在事实文本高度重叠。

**具体操作**：
- 提取每条条目的"事实"字段做相似度比较
- 标题 + 事实组合相似度 > 70% 判定为 DUPLICATE
- 跨文件比较，不只是文件内

**示例**：
- `_global.md/G-005: 用 HikariCP` 与 `infra.md/M-002: 连接池统一用 HikariCP` 是 DUPLICATE
- 建议保留更具体（带"为什么"、带行号）的那一条

## 报告组织

按**优先级** + **文件**双重组织：

### 优先级映射

| 问题 | 优先级 |
|---|---|
| STALE_CODE | 🔴 高 |
| STALE_TIME_SEVERE | 🔴 高 |
| DUPLICATE | 🔴 高（如果两条都还在生效）/ 🟡 中（如果只是有点重） |
| STALE_CODE_PARTIAL | 🟡 中 |
| STALE_TIME | 🟡 中 |
| DERIVABLE | 🟡 中 |
| INCOMPLETE_WHY | 🟢 低 |

### 每条问题输出格式

```
- <ID> [<问题代码>] <一句话说明问题>
  建议：<具体处置建议，引用具体命令>
```

例：

```
- M-003 [STALE_CODE] 关联代码 src/auth/legacy.ts 已不存在
  建议：/forget M-003，或如果代码迁移到新位置，用 /update-memory M-003 改路径
```

---

## 不做的事

- ❌ 不要自动删除或修改任何条目（原则 9：用户驱动）
- ❌ 不要把 DERIVABLE 标记当强制——判断保守，拿不准就跳过
- ❌ 不要审计 AGENTS.md 的固定 4 段结构
- ❌ 不要在没有任何问题时编造问题为了"看起来有用"

---

## 输入/输出契约

被 `/audit-memory` 命令调用时：

**输入**：
- 范围（全量 / 指定文件）
- 当前日期

**输出**：
- 结构化的问题列表，按优先级 + 文件分组
- 每个问题：条目 ID、问题代码、说明、建议
- 汇总：总条目数、各优先级问题数
